Classier Classes for JavaScript¶

Wooya. You love JavaScript and use jQuery or another lightweight library extensively but you notice that your code becomes messier and messier the larger the application grows? Well, that’s where Classier comes in. Classier is a small JavaScript library that implements classes and private/protected class members (properties and methods) for JavaScript.

Classier is partly inspired by, and uses the same API/syntax as, the excellent JavaScript library Classy (but does not share its codebase).

Classier also borrows large parts of its documentation from Classy. The biggest difference between the two libraries is that you can use private and protected class members with Classier. Also class constructors get proper names in Classier, thus allowing for easier debugging - most web browsers will console.log the name of a class when you look at an instance of it.

Another (small, but sometimes important) difference between Classy and Classier is that Classier can use arrays and objects as property values in a class declaration and they will be cloned (rather than referenced) when you create a class instance.

Classier started its life as a small library called Object.createclass and therefore still supports an older, alternative API/syntax.

Where can you get Classier? Here: github: Classier 1.5

How do you use Classier?

Enabling The Library¶

<script type="text/javascript" src="classier.js"></script>

Defining Classes¶

var Animal = Class.$extend({

  // __init__ will run each time we create an instance 
  __init__ : function(name, age) {
    // properties that we need to set initially
    // need only be defined inside __init__
    this.name = name;
    this.age = age;
  },

  // but other properties can be defined here
  health: 100,
  _secret: "I'm always hungry.",             // Protected (starts with _ )
  __extraSecret: "I'm scared of humans.",    // Private (starts with __ )

  // as well as methods
  die : function() {
    this.health = 0;
  },

  eat : function(what) {
    this.health += 5;
  },

  tellSecret : function() {
    return this._secret;
  },

  tellExtraSecret : function() {
    return this.__extraSecret;
  }

});

Creating Subclasses¶

var Tiger = Animal.$extend({

  eat : function(animal) {
    this.$super(animal);
    animal.die();
  }

});

var Sheep = Animal.$extend({

  __init__ : function(name, age, shorn) {
    this.$super(name, age);
    this.shorn = shorn;
  }

});

Using These Classes¶

var leo = Tiger("Leo", 3);
var molli = Sheep("Molli", 1);
leo.eat(molli);

Alternatively you can also use the new operator like with regular JavaScript classes. But no worries, both do exactly the same:

var leo = new Tiger("Leo", 3);
var molli = new Sheep("Molli", 1);
leo.eat(molli);

You want to check if an object is an instance of something? That’s easy as well:

var leo = new Tiger("Leo", 3);

leo instanceof Animal // -> true

leo.constructor === Animal // -> false
leo.constructor === Tiger // -> true

Protected and private class members¶

Protected and private class members (properties and methods) can only be accessed from within the methods of a class instance. Protected members are inherited by subclasses, private members are not inherited by subclasses.

  • All class members with names starting with an underscore, "_", become protected.
  • All class members with names starting with two underscores, "__", become private, except those that also end with two underscores (like __init__ , a public method).
var ani = new Animal("Annie", 3);

ani._secret  
// -> ReferenceError: You can not access the protected property _secret

ani.__extraSecret
// -> ReferenceError: You can not access the private property __extraSecret

ani.tellSecret()  
// -> "I'm always hungry."

ani.tellExtraSecret()
// -> "I'm scared of humans."


var leo = new Tiger("Leo", 3);

leo._secret  
// -> ReferenceError: You can not access the protected property _secret

leo.__extraSecret
// -> undefined

leo.tellSecret()  
// -> "I'm always hungry."

leo.tellExtraSecret()
// -> undefined

Recap and extra features¶

What you can do with Classier:

Class.$classierVersion¶

The version of Classier as a string (1.5 for instance).

Class.$classyVersion¶

The version of Classy we are compatible with as a string (1.4 for instance).

Class.$extend(properties)¶

Creates a new class by extending the given class. The properties is an object with class attributes and methods (functions) for this class.

Class.$withData(data)¶

Creates a new instance of the class by bypassing the constructor and assigning the attributes from the given object. This is useful if you want to attach methods to an object recieved from a JSON object:

var MyClass = Class.$extend({
  __init__ : function() { alert('called'); },
  toString : function() {
    return this.value;
  }
});
var obj = MyClass.$withData({value: 42});
alert(obj.toString());

The example above will alert "42" only. So the constructor is not called but you have access to the methods defined in the class.

Class.$noConflict()¶

Removes the Class object from the window object and restores what was there before Classier was loaded. It then returns the class object from the function. This makes it possible to use multiple versions of Classier side by side or in combination with other libraries that also define an object with that name. Example usage:

(function(Class) {
  // here you can use the Classier Class object, outside it
  // won't be available
})(Class.$noConflict());
this.$super()¶

When called from within a Classier function this invokes the parent function of the same name. The arguments are forwarded directly. Check out the examples above for how that works.

this.$class¶

The class object for this instance. This is especially useful to acccess class attributes from instances. (Same as writing this.constructor)

Class.__init__()¶

The constuctor of a class. This is created immediately after a class was created and gets all the arguments passed when the object was created:

var leo = new Tiger("Leo", 3);
// --> __init__ is invoked with "Leo" and 3
Class.__include__¶

When this attribute is provided when the class is created and set to a list of ordinary objects, each attribute of such an object is mixed into the class. Imagine you have some functionality you want to export from multiple classes, this is the perfect way:

var UpdateMixin = {
  update : function(options) {
    for (var key in options)
      this[key] = options[key];
  }
};

var MyClass = Class.$extend({
  __include__ : [UpdateMixin]
});

var obj = MyClass();
obj.update({foo: 1, bar: 2});
// obj.foo --> 1

This is similar to subclassing but has the advantage that you can mix multiple functions in at the same time. Please note that you can only mix in regular objects, not actual classes with a prototype (like those created with Class.$extend).

Class.__classvars__¶

An object of items that should be added to the class as class level attributes:

var MyClass = Class.$extend({
    __classvars__ : {
      MAX_ITEMS : 42
    }
});
alert(MyClass.MAX_ITEMS);

This shows a message box with "42". To access these attributes from an instance, use the $class attribute of instances:

alert(MyClass().$class.MAX_ITEMS);

Alternate/legacy syntax¶

Because Classier started its life without having glanced at its cousin library Classy you can also use an alternate syntax to create your classes. It involves using Object.createClass to create a class and the "magic property" _extends to specify inheritance.

Also: The magic method init is used instead of __init__ and instead of this.$super you write $this._super.

And _class used to be necessary to give the class constructor a proper name (thus making it easy to see what class an object belongs to when debugging). Since version 1.5 this is taken care of automatically and the class name will be that of the variable you assign the class to.

This syntax can still be used in parallell with the newer syntax. So we can write:

var Sheep = Object.createClass({

  _class : "Sheep", // not needed since Classier 1.5

  _extends: Animal,

  init: function(name, age, shorn) {
    this._super(name, age);
    this.shorn = shorn;
  }

});

and it would mean just the same as:

var Sheep = Animal.$extend({

  __init__ : function(name, age, shorn) {
    this.$super(name, age);
    this.shorn = shorn;
  }

});
